Manual de Usuario: Módulo de Despacho

Salida directa e inmediata de inventario para Socios Comerciales (Otros Hortofrutícolas), incluyendo el nuevo flujo de SubClientes para Fall Creek.

Manual Socios Comerciales Completo

1. Introducción y Conceptos Clave

La pestaña Despacho del módulo de Socios Comerciales (Otros Hortofrutícolas) gestiona la salida física de fruta de terceros (ej. plantas de Fall Creek o arándanos) almacenada en las cámaras de frío de la planta.

Rebaja Directa vs Flujo de Cerezas
A diferencia del flujo de cerezas (el cual genera una solicitud en espera que requiere la validación y picking posterior de los grueros), el despacho de Socios Comerciales realiza una rebaja directa e inmediata del stock en la base de datos en el instante en que el operador confirma la transacción. No existen estados pendientes intermedios.

Descripción de los Campos:

  • Cliente (Obligatorio): Permite seleccionar la entidad dueña del stock. Para evitar pantallas vacías, el sistema solo lista aquellos clientes de tipo "FRUTA" que tengan al menos 1 bin o pallet en estado "Almacenado" y con cantidades mayores a cero.
  • Filtrar por Lote: Permite ingresar parte del lote del sistema (ej. LOT-FC-123) o lote del cliente para aislar rápidamente la variedad o el cargamento específico que se desea retirar.
  • Documento de Despacho (Obligatorio): Campo numérico o de texto donde se digita la guía de despacho física que acompaña al camión. El sistema mantendrá deshabilitado el botón de confirmar hasta que se ingrese este número.

3. Selector de SubCliente (Caso Especial: Fall Creek)

La última actualización incorpora un campo condicional y obligatorio denominado "SubCliente", aplicable exclusivamente cuando el cliente seleccionado es Fall Creek (código EXP004).

Regla de validación de Fall Creek
Si se selecciona a Fall Creek como el exportador y se intenta confirmar un despacho sin elegir un SubCliente, el sistema bloqueará la operación inmediatamente y mostrará un mensaje emergente de alerta:
"Debe seleccionar un SubCliente para despachar a Fall Creek."

Funcionamiento del Selector:

  1. Filtro Automático: El listado de subclientes se genera automáticamente en tiempo real. Filtra a los productores del sistema que cumplen con dos condiciones simultáneas:
    • Su estado es activo en la base de datos.
    • Tienen asignado o vinculado el exportador EXP004 (Fall Creek) en su ficha maestro de socios comerciales.
  2. Identificación Visual: El selector muestra el nombre abreviado (shortName) si existe, o el nombre completo del productor para facilitar la búsqueda al operador.
  3. Destino de Datos: Al seleccionar un subcliente, el sistema asocia de forma permanente el despacho a este productor. Esto se registra en la base de datos en los campos destinationClientName y destinationClientRUT para auditorías de destino.

4. Acordeón de Lotes y Desglose de Ubicaciones

Al ingresar el cliente, se despliega una lista en formato de acordeón agrupando el stock por su identificador de lote de sistema y lote cliente. Al hacer clic en un acordeón se abre la tabla detallada del lote:

SELECCIÓN COORDENADA PRODUCTO (VARIEDAD) LOTE CLIENTE DISP. A DESPACHAR
C3 - Nivel 2 Sekoya Crunch LOTE-FC-MAYO-26 12 Bins
C5 - Nivel 1 Sekoya Grande LOTE-FC-MIXTO-A 8 Bins

Columnas Clave:

  • Ubicación Física (Coordenada): Muestra la celda exacta de la cámara donde se localiza el producto (ej. C3, C5).
  • Disp. (Disponible): Representa el saldo total actual registrado de ese lote en esa coordenada física exacta.
  • A Despachar: Campo numérico editable donde se define la cantidad a retirar del inventario. Por defecto, al marcar la casilla de selección, el sistema llena este campo con el total disponible (despacho total).

5. Cómo Realizar un Despacho Parcial

El sistema soporta el retiro fraccionado de stock en una misma coordenada para adaptarse a la capacidad del camión o solicitudes especiales del cliente. Para realizar un despacho parcial, siga estas instrucciones:

  1. Haga clic en la casilla de selección de la fila correspondiente a la coordenada. El campo A Despachar se completará automáticamente con el stock máximo disponible.
  2. Haga clic sobre el campo numérico de la columna "A Despachar".
  3. Reemplace el valor sugerido por la cantidad real que se subirá al transporte (ej: si hay 12 disponibles, ingrese 5).
  4. El sistema calculará automáticamente la suma final en la barra inferior (ej. "Total a despachar: 5 Bins").
  5. Al presionar "Confirmar Despacho", el sistema descontará únicamente 5 unidades de la coordenada seleccionada. La base de datos mantendrá los 7 bins restantes en la cámara en perfecto estado Almacenado para futuras operaciones.

6. Registro Técnico e Impacto en Firestore

Al presionar el botón "Confirmar Despacho" en la barra inferior, el sistema procesa una transacción por lotes (Write Batch) en la base de datos de FriO que ejecuta los siguientes cambios de forma segura:

1. Colección otherFruitReceptions (Recepción Origen)

Se localiza el documento de ingreso del lote y se actualiza la cantidad disponible del ítem en la coordenada restando el valor ingresado. Si la cantidad llega a 0, el ítem permanece en el historial pero con stock nulo.

2. Colección otherFruitMovements (Historial de Transacciones)

Se crea un nuevo documento de movimiento con la siguiente estructura de campos (con soporte para subcliente):

Campo Firestore Tipo Descripción
type string Tipo de movimiento. Siempre se registra como 'salida'.
clientId string Identificador del socio comercial principal (ej. EXP004).
clientName string Nombre del socio comercial principal (ej. Fall Creek).
document string Número de guía de despacho física ingresado por el operador.
destinationClientName string Nombre del SubCliente receptor (solo Fall Creek). Ej: Agrícola El Vergel.
destinationClientRUT string RUT del SubCliente receptor (solo Fall Creek). Ej: 77.654.321-K.
createdAt timestamp Fecha y hora del servidor al momento del despacho.
items array Lista de los ítems despachados, incluyendo código, nombre del producto, lote cliente y cantidad.
Importante
Estos movimientos son irreversibles de forma automática desde la interfaz del operador. Cualquier anulación de despacho o error en la digitación del número de guía o subcliente debe ser corregida o auditada por un administrador del sistema directamente en el panel de control o colecciones del Kardex.
